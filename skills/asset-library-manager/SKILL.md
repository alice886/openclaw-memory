# Asset Library Manager v3.0

**描述**: 智能资产库管理系统，支持三重过滤（文件格式/文件名/图片内容）、语义标签生成、自然语言搜索和精准匹配。

**职责范围**:
- ✅ 资产文件扫描与三重过滤
- ✅ 语义标签生成（AI 理解图片内容）
- ✅ 自然语言语义搜索
- ✅ 资产索引生成与维护
- ✅ 项目完整性检查
- ✅ 上传资产到 ZenStudio 媒资库

**不负责**:
- ❌ AI 生成任务执行（由 `ai-shot-generator` 负责）
- ❌ 分镜脚本解析（由 `pdf-storyboard-parser` 负责）

**触发条件**: 
- 用户提到"扫描资产"、"查找资产"、"初始化资产库"
- 用户搜索角色/场景/道具（如"找青年刀马"）
- 用户以自然语言描述图片内容（如"找一张沙漠黄昏不带人的图片"）
- 用户要求"检查项目完整性"
- 用户需要"上传资产到云端"

## 📁 文件结构规范

```
<项目路径>/
├── assets/                    # 资产库根目录
│   ├── characters/           # 人设图
│   ├── scenes/               # 场景图
│   ├── props/                # 道具图
│   └── moods/               # 氛围图
└── project_index.json        # 资产索引（含语义标签）
```

## 🔧 核心功能

### 1. 初始化资产库
**命令**: 
```
初始化资产库 <项目路径>
```

**执行流程（三阶段）**:

#### Phase 1：三重过滤扫描
1. 扫描 `assets/` 各子目录
2. **Step 1**：文件格式过滤（排除 PSD/PSB/AI 等）
3. **Step 2**：文件名关键词过滤（排除 RR/线稿/sketch 等）
4. **Step 3**：图片内容分析（`image` 工具逐张验证是否为上色稿）

#### Phase 2：语义标签生成（新增）
5. **对每个通过过滤的资产**，用 `image` 工具分析图片内容
6. 生成语义标签写入 `project_index.json`

#### Phase 3：索引输出
7. 输出统计报告

**Phase 2 语义分析 Prompt**:
```
分析这张图片，用简短的自然语言描述其内容。

输出 JSON 格式：
{
  "description": "一句话描述这张图的内容",
  "tags": ["关键词1", "关键词2", "关键词3"...],
  "scene": "场景类型",
  "has_people": true/false,
  "has_character": "角色名或null",
  "mood": "氛围/情绪",
  "time_of_day": "时间/时段"
}

标签生成规则：
- scene: 如"沙漠"、"竹林"、"大殿"、"街道"等
- has_people: 是否有明显人物（true/false）
- has_character: 如果能识别出具体角色名则填入，否则填null
- mood: 如"紧张"、"温馨"、"萧瑟"、"激烈"等
- time_of_day: 如"白天"、"夜晚"、"黄昏"、"清晨"等
- tags: 综合所有维度，所有能用上的关键词

示例输出：
{
  "description": "竹林深处的激烈打斗场景，青年武士持刀对峙",
  "tags": ["竹林", "打斗", "持刀", "古风", "动态", "紧张", "白天", " outdoors"],
  "scene": "竹林",
  "has_people": true,
  "has_character": "青年刀马",
  "mood": "紧张",
  "time_of_day": "白天"
}
```

**语义标签示例（project_index.json 中的结构）**:

### 2. 语义搜索（核心新增功能）
**命令**:
```
找一张沙漠黄昏不带人的图片
搜索"竹林打斗武士"
查找有的人物是刀马的场景
搜一个室内晚上的场景
```

**执行流程**:

#### Step 1：解析自然语言 → 搜索条件
将用户的自然语言描述转换为搜索条件：

```
"沙漠黄昏不带人"
→ {
    scene: "沙漠",
    time_of_day: "黄昏",
    has_people: false
  }

"竹林激烈打斗"
→ {
    tags包含: ["竹林", "打斗"],
    mood: "激烈"
  }

"找夜晚有刀马的场景图"
→ {
    has_character: "刀马",
    time_of_day: "夜晚"
  }
```

#### Step 2：标签匹配搜索
在 `project_index.json` 中按以下优先级匹配：

**优先级 1：精确属性匹配**
- `has_people = false` 且 `scene = 沙漠` → 直接命中

**优先级 2：Tags 关键词匹配**
- 用户的每个关键词在资产的 `tags` 数组中匹配
- 匹配率 = 命中的标签数 / 用户关键词总数

**优先级 3：模糊场景匹配**
- `scene` 字段模糊匹配（如"竹林"匹配"竹林深处"）

#### Step 3：排序与输出
- 按匹配率排序
- 匹配率相同则按 `priority` 字段排序
- 展示前 5 个结果供用户选择

**搜索结果示例**:

## 🔍 三重过滤规则

### Step 1：文件格式过滤
排除：`.psd .psb .ai .sketch .blend`

### Step 2：文件名关键词过滤
排除关键词：
- 线稿 / lineart / line-art / sketch / draft / rough / 草图 / 轮廓 / outline
- 未上色 / uncolored / bw / black-white
- RR（Rough Rendering = 线稿）
- 上色中 / coloring / 描线

可用标记：
- 清稿 / 色稿 / 上色 / final / v2 / v3 / 三视图 / turnaround

### Step 3：图片内容分析
使用 `image` 工具验证：

**判定标准**：
- ❌ 线稿：大量清晰线条 + 无阴影 + 无质感（纯色平涂也算）
- ✅ 上色稿：有颜色 + 有阴影/质感/立体感

> ⚠️ 赛璐璐风格（有颜色但几乎无阴影）容易被误判。
> 关键看**有无阴影和立体感**，而非有无颜色。

## 🛠️ 实现要点

### 搜索判断逻辑
```
用户输入：找一张竹林黄昏不带人的图片
    ↓
检测关键词：竹林、黄昏、不带人
    ↓
竹林 → scene 字段
黄昏 → time_of_day 字段
不带人 → has_people = false
    ↓
搜索 assets.scenes 中满足：
  scene包含"竹林" AND time_of_day="黄昏" AND has_people=false
    ↓
返回匹配结果
```

### 搜索条件组合规则
- AND 关系：同一维度的条件（scene + time_of_day）
- OR 关系：同一维度的多值（scene="竹林" OR scene="森林"）
- 负向排除：不带人 → has_people = false

### 性能优化
- Phase 1 过滤（文件名）：快，毫秒级
- Phase 2 标签生成（AI 分析）：慢，每张图约 2-3 秒
  → 支持增量更新：已有标签的资产跳过分析
  → 建议在初始化时一次性完成
- 搜索匹配：快，毫秒级（内存操作）

### 增量更新语义标签
```
命令：更新资产标签 <路径>
```
只对指定路径的资产重新分析，不影响其他资产。

## 📌 常见问题

### Q1: 语义搜索搜不到怎么办？
- 尝试不同的描述方式（"黄昏" vs "日落" vs "傍晚"）
- 放宽条件后重试
- 检查 `project_index.json` 中该资产的 tags 是否覆盖了你的搜索词

### Q2: 语义标签不准怎么办？
- Agent 生成标签后可手动修正 `project_index.json`
- 使用"更新资产标签 <路径>"命令重新分析

### Q3: 资产更新后标签过期？
- 新增资产：运行"更新资产标签 <新资产路径>"
- 修改资产：运行"更新资产标签 <资产路径>"重新分析
